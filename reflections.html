Lessons Learned

Early prototype feedback is crucial for identifying usability issues such as missing system feedback and navigation controls. Collaborating with peers for heuristic evaluation revealed blind spots that individual design can miss.

Balancing minimalism with functionality taught us to prioritize essential features and remove unnecessary complexity while still guiding users effectively.

What Worked Well & Improvements

  • Worked Well: Consistent layouts across pages created a cohesive experience. The dark-mode design offered strong visual contrast and a modern aesthetic.
  • Could Be Improved: Interactive elements (buttons, back links) need implementation early. The color palette needs refinement to improve accessibility and support both dark and light preferences.

Prototype Alignment & Heuristic Impact

  • User Needs: The high-fidelity prototype addresses target users by presenting clear pathways for education, outreach, and job services in a straightforward interface, reducing cognitive load for users with limited digital literacy.
  • Heuristic Evaluation Impact: Heuristic feedback led to adding emergency-exit controls, improving error-prevention flows, and planning for more informative system-status messages—ensuring the final design better supports user control, error recovery, and overall learnability.
